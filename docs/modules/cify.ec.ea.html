Evolutionary Algorithm#

An evolutionary algorithm (EA) is a generic population-based metaheuristic algorithm. Evolutionary algorithms are a subset of evolutionary computation, however, forms a collection of other subsets of common metaheuristics. For example, genetic algorithms (GAs), genetic programs (GPs) and evolutionary programs (GPs) are all subsets of evolutionary algorithms.

The evolutionary algorithms contained in CIFY are easily customizable by changing the components used by any of the algorithms or their parameters. All algorithms have default operators and parameters and can be used as they are, however, to improve performance on your chosen objective functions, it is best to try different operators and control parameters for these operators.

In this package, the generic EA is given, which does not make use of mutation. If you are looking for a comparable Algorithm that does, try the genetic algorithm, GA.

__init__(obj_func: cify.core.objective_function.ObjectiveFunction, populations: Optional[List[cify.core.base_classes.collection.Collection]] = None, n_individuals: Optional[int] = None, crossover_operator=None, crossover_params: Optional[dict] = None, selection_operator=None, selection_params: Optional[dict] = None, **kwargs)#
Parameters

  • obj_func (ObjectiveFunction) – The objective function to optimize.

  • populations (list of Collection objects, optional) – A list of populations to use to optimize the ObjectiveFunction, defaults to a single population of 50 Individual s.

  • n_individuals (int, optional) – The number of individuals to use in the population, defaults to 50

  • crossover_operator (function, optional) – The operator to use for crossover, defaults to discrete uniform crossover.

  • crossover_params (dict, optional) – Additional parameters for the crossover operator.

  • selection_operator (function, optional) – The operator to use for selection, defaults to random selection.

  • selection_params (dict, optional) – Additional parameters for the selection operator.
